# モデル化とシミュレーション体験ツール

## 背景

本 Web アプリケーションは高等学校の「情報Ⅰ」を対象とし、「モデル化とシミュレーション」を体験的に学習できる環境を意図しています。

## 技術的要件

- Next.js (Pages Router) で実装されています。
- TypeScript を使用しています。
- 静的にホスティングできるように設計されています。
- スマートフォンでも使用できるレスポンシブデザインに対応しています。
- クイズの部分や、シミュレーションを体験できる部分は、なるべく実装が共通になるようコンポーネント化されています。それぞれのコンポーネントでのみ使用するスタイルは `*.module.css` に記述します。
- 画像素材は必要に応じて生成 AI で作成します。

## ページ構成

### トップページ

トップページでは、教科書のような丁寧な説明が記載されています。

まず、「モデル」「シミュレーション」という用語の意味を説明しています。「モデル」とは、実際の物や事象について、簡略化して表現したものです。シミュレーションとは、モデルを使って事象を試行することです。

また、モデルは「静的モデル」「動的モデル」に分けることができ、さらに「動的モデル」は「確定的モデル」「確率的モデル」に分けることができることをそれぞれ説明しています。静的モデルは時間の経過を扱わないモデル、動的モデルは時間の経過を扱うモデルです。また、確定的モデルは不確実な要素を含まないモデル、確率的モデルは不確実な要素を含むモデルです。これらは樹形図のような構成で整理されており、「静的モデル」「確定的モデル」「確率的モデル」をクリックすることで、それぞれのページに遷移できるようになっています。

### 静的モデルのページ

ページの上部では、改めて「静的モデル」について説明があります。その下に、2 つのモデルのクイズが用意されています。

#### 教室の座席表

教室内で子どもたちが着席している手書きイラストが表示されており、これをモデル化するとどうなると思うかというクイズが紹介されています。クイズの答えは最初隠されており、アニメーションでときどき揺れ動いていて、これをクリックすると答えである座席表の手書きイラストと解説が表示されます。最終的に両方のイラストが横並びに表示され、矢印で結ばれた状態になります。

#### ミニカー

背景を含むリアルな自動車の写真が表示されており、これをモデル化するとどうなると思うかというクイズが紹介されています。クイズの答えは最初隠されており、アニメーションでときどき揺れ動いていて、これをクリックすると答えであるミニカーを手に持った写真と解説が表示されます。最終的に両方のイラストが横並びに表示され、矢印で結ばれた状態になります。

### 動的モデル（確定的モデル）のページ

ページの上部では、改めて「動的モデル」「確定的モデル」について説明があります。その下に、2 つのシミュレーションが用意されています。

#### 水槽のシミュレーション

水槽に一定のペースで水を加えていくシミュレーションです。

水槽の大きさ（L）、最初から水槽に溜まっている水の量（L）、1秒ごとに加える水の量（L/秒）を設定できます。

1秒進めるボタンが用意されています。

各フレームにおける水槽と水量は canvas で可視化されます。水槽から水が溢れたときは、水槽の周囲が水浸しになります。

描画は上書きされるのではなく、横方向にフレーム数が増えていき（画面幅を超えた場合は横スクロール可能）、時間の経過に伴う変化の仕方を後から確かめることができます。

#### 利息のシミュレーション

預金の利息が増えていくシミュレーションです。複利です。

最初の預金額（円）、年利率（%）を設定できます。

1年進めるボタンが用意されています。

各フレームにおける預金額は canvas で可視化されます。それぞれの金額のお札と小銭として描画されます。お札と小銭には金額が表記されています。お金は万の位、千の位、百の位、十の位、一の位の行で構成されます。例えば千の位は、0 枚以上 1 枚以下の五千円札と、0 枚以上 4 枚以下の千円札で表現されます。同じ種類のお札は重ねて少しずらすことで限られた描画スペースで表現します。なお、一万円札の枚数が非常に多い場合、overflow: hidden とします。

描画は上書きされるのではなく、横方向にフレーム数が増えていき（画面幅を超えた場合は横スクロール可能）、時間の経過に伴う変化の仕方を後から確かめることができます。

### 動的モデル（確率的モデル）のページ

ページの上部では、改めて「動的モデル」「確率的モデル」について説明があります。その下に、2 つのシミュレーションが用意されています。

#### すごろくのシミュレーション

さいころを振って人型の駒を進めていくすごろくのシミュレーションです。

マス数を指定できます。ただし、マスをはみ出して、もっと盤面が大きかった場合にマスがある場所へ、どんどん駒は進んできます。

1ターン進めるボタンが用意されています。

各フレームにおけるさいころの出目とすごろくの盤面は canvas で可視化されます。また、そのフレームで進んだマスは色が変わっており、差分がわかりやすいようになっています。さいころの出目はイラストで表示し、最初のフレームでは表示しません。

描画は上書きされるのではなく、横方向にフレーム数が増えていき（画面幅を超えた場合は横スクロール可能）、時間の経過に伴う変化の仕方を後から確かめることができます。

#### 待ち行列のシミュレーション

券売機の前に並ぶ人の待ち行列のシミュレーションです。

各秒に新しく券売機の前に現れる人数の確率（%）を指定できます。また、それぞれの人が券売機で用を済ませるのにかかる時間（秒）を指定できます。

1秒進めるボタンが用意されています。

各フレームにおける券売機の前の待ち行列は canvas で可視化されます。それぞれの人は異なる色の帽子を被っており、区別が容易になっています。先頭の人の頭の上には、券売機前での経過時間が帯グラフのようなゲージで表されます。また、ルーレットのような円形とそのうちの確率を表す扇形の表示で、そのフレームにおいて券売機の前に新しく人が現れたかどうかを確認できます。新しく人が現れた場合は、ルーレットとその人が同じ色の太線で強調して表示されます。ただし、ルーレットは最初のフレームには表示しません。

描画は上書きされるのではなく、横方向にフレーム数が増えていき（画面幅を超えた場合は横スクロール可能）、時間の経過に伴う変化の仕方を後から確かめることができます。

## 共通コンポーネントの細かな仕様

- フレームが横並びになっている箇所について、フレームを追加時に横スクロールが発生する場合は、自動的に右横に目いっぱいスクロールするようにします。
- 画面上は「フレーム」とは呼ばず、「x 秒後」など、それぞれの単位で表示します。
